# 交付文件：AI 健身追蹤 App（核心閉環）

> 給使用 Claude Code 的開發者：這份文件描述一套健身 App 的 UI/UX 設計。讀完這份 README 就能在你的程式碼環境中完整實作，不需參與當初的設計討論。

---

## 一、Overview（這是什麼）

一套以 **AI 排課 + 訓練追蹤 + 計畫 vs 實際記錄**為核心的健身 App。核心使用閉環：

```
輸入身體資料 → AI 排課表 → 進健身房跟著清單練（記錄每組 + 組間休息計時）
→ 看數據 → AI 週期回顧 → 調整課表 → 循環
```

設計聚焦在「健身房當下」的訓練體驗：大按鈕、單手可操作、抬頭就懂、休息計時為主角。

### 字體
- **數字 / 拉丁字**：`Space Grotesk`（weights 400/500/600/700）。所有重量、次數、計時、統計數字都用它，並加 `font-variant-numeric: tabular-nums` 讓數字等寬。
- **中文**：`Noto Sans TC`（weights 400/500/700/900）。標題用 900，內文 400/500。
- **手寫鼓勵語**（如「做得好，今天也有進步」）：`Caveat`（選用，可用一般字體代替）。

### 字級（px）
| 用途 | 大小 / 重量 |
|---|---|
| 大計時數字 | 60 / 700（Space Grotesk）|
| 畫面大標題（中文 900）| 22–26 |

| 內文 | 12–13 / 400–500 |
| 標籤 / 說明 | 10–11 / 400 |

### 間距 / 圓角 / 陰影
- 頁面左右 padding：`22–24px`
- 卡片內 padding：`13–18px`
- 元素間距：用 flex/grid `gap`，常用 `9–14px`
- 圓角：按鈕（pill）`23–29px`、卡片 `15–20px`、輸入框 `10–13px`、彈出層上緣 `28px`、手機外框 `40px`
- 主按鈕陰影：`0 9px 22px rgba(95,227,154,.3)`
- 卡片陰影：一般無，手機外框 `0 18px 48px rgba(0,0,0,.32)`

## 五、Screens / Views（畫面清單）

App 分三大區塊 + 底部分頁。以下逐一描述。完整可點擊版本見 `完整原型.dc.html`。

### 區塊 A — Onboarding（首次設定，5 步）

**1. 歡迎頁**
- 佈局：垂直置中。72px 圓角方形 Logo（綠漸層，內含字母）→ 主標（中文 900, 26px）→ 副標（弱文字）；底部固定主 CTA「開始設定」+ 文字連結「已經有帳號？登入」。
- 背景：深色綠暈漸層。

**2. 基本資料**
- 頂部 3 段進度條（已完成段 `#5fe39a`，未完成 `#26241d`）。
- 表單欄位用「大數字 + 底線」樣式（非傳統輸入框）：身高/體重/年齡顯示為大 Space Grotesk 數字，下方底線；當前聚焦欄位底線為綠色。性別為兩顆膠囊選擇（選中＝綠框＋淡綠底）。
- 底部 CTA「下一步」。

**3. 目標**
- 進度條 2/3。三張選項卡（增肌/減脂/維持），每張：左側 44px 圓角圖示、標題＋說明、右側選中打勾。選中卡＝淡綠底＋綠框＋綠漸層圖示。

**4. 訓練頻率**
- 進度條 3/3。「一週幾天」為 5 顆圓鈕（2–6），選中＝綠漸層＋發光。下方動態提示（如「4 天 → 建議上下肢分化」）。「每次多久」三顆膠囊（45/60/90 分）。
- 底部 CTA「生成我的課表 ✨」。

**5. AI 生成中**
- 置中：120px 進度環（綠，drop-shadow 發光）內含 ✨。標題「AI 正在為你排課表」＋說明。下方逐項勾選清單（分析身體資料 ✓ / 規劃訓練分化 ✓ / 挑選每日動作…）。
- 約 2–3 秒後自動跳到課表確認頁。

### 區塊 B — AI 課表確認 + 首頁

**6. 課表確認頁**
- 頂部「✨ AI 已排好」膠囊標籤 + 「可修改」。標題「本週課表」+ 說明「點動作可調整」。
- 第 1 天卡片（今日漸層底）展開顯示動作清單，每列：動作名 + 「3×8 · 休90s」+ **綠色「改」chip**（點擊開編輯彈出層）。其他天為收合列（右側 chevron）。
- 底部兩鈕：「重新生成」（次要，邊框）+「確認並儲存」（主要，綠漸層，較寬）。
- **這是核心慣例：AI 預填 → 使用者過目修改 → 存檔。** 同一套「確認介面」語言也用在未來的器材辨識、InBody 匯入。

**7. 編輯動作彈出層（bottom sheet）**
- 從底部滑入，深色卡（`#1b1914`），頂部拖曳條。
- 標題＝動作名 + 「調整組數 / 次數 / 休息 · 會套用到訓練」。
- 三個步進器（組數 / 次數 / 休息秒數）：每個是 `−  數字  +`，數字 Space Grotesk 700；「−」弱色、「+」綠色。
- 底部「完成」綠漸層按鈕。儲存後寫回課表資料（會影響之後的訓練）。

**8. 首頁 Dashboard**
- 頂部問候（日期 + 「早安，阿凱 👋」）+ 右上頭像（綠漸層圓，點擊到個人頁）。
- **最大的綠色「今日訓練」卡**（今日漸層底）：標籤「今日訓練」+ 預估時間 + 訓練名（「推日 · 胸肩三頭」）+ 說明 + 「開始訓練」綠漸層按鈕。
- 下方兩張小統計卡：體重（72.4kg ↓0.6）、本週訓練（2/4 次 + 進度點）。
- 底部 5 分頁導覽。
- **休息日變體**：問候改「休息日 😌」，主卡改恢復訊息 + 「查看本週課表」，並顯示體重趨勢迷你折線圖。

### 區塊 C — 訓練追蹤主場景（健身房當下，最重要）

採用「**單動作聚焦 + 計時為主**」的佈局（B+C 方向）。

**9. 開始前**
- 頂部「‹ 返回」。標題「推日 · 胸肩三頭」+ 「5 個動作 · 預估 55 分鐘 · 點動作可調整」。
- 動作清單：每列 = 編號（綠字）+ 動作名 + 「3 組 × 8 下 · 60kg」+ 綠色「改」chip（點擊開編輯彈出層，同 #7）。
- 底部大 CTA「開始今天的訓練」。

**10. 記錄這一組**（極簡風，參考 Graphite 方向）
- 頂部資訊列：綠色「推日」膠囊 + 「動作 2 / 5」+ 「⏱ 18:24」（總計時）。
- 動作名（中文 700, 26px）+ 「目標 3×10 · 課表 22kg」。
- 組進度條（每組一段，已完成綠）+ 「SET 2 / 3」。
- 主體：兩列大型步進器（重量、次數），樣式為 `標籤 ……  −  大數字  +`，數字 34px。「−」弱色、「+」綠色。重量步進 2.5kg、次數步進 1。
- 底部「完成這組」綠漸層大按鈕（高 56px）。
- **打勾後自動進入休息倒數。**

**11. 休息倒數**（全螢幕，深色綠暈背景，計時為主角）
- 頂部資訊列同上。
- 中央「休 息 中」標籤（綠，字距 5）+ **222px 發光計時環**（綠漸層描邊 + drop-shadow 發光，`stroke-dashoffset` 隨剩餘秒數動態），環內大數字（60px）+ 「建議 90 秒」。
- 環下方鼓勵語（每個動作輪換不同句）。
- 「下一組」資訊卡（動作名 + 第幾組）。
- 兩鈕：「跳過」「+30s」（邊框次要按鈕）。
- 底部小字：「倒數結束震動提示 · 鎖屏仍通知」。
- 倒數歸零或按跳過 → 自動進下一組（回 #10）；全部完成 → 訓練完成。
- **實作注意**：計時須能背景運作、鎖屏到時仍發本地通知；用震動而非聲音（健身房吵）。

**12. 訓練完成**
- 置中：72px 綠光圓 ✓ + 「訓練完成！」+ 鼓勵語。
- 兩張統計卡：總時間、完成組數。
- 「計畫 vs 實際」清單：每個動作顯示實際做的重量×次數。
- 底部「完成並存檔」→ 回首頁。

**13. 課表 tab（本週課表，可展開）**

## 六、Interactions & Behavior（互動與行為）

- **導覽流程**：歡迎 → 基本資料 → 目標 → 頻率 → AI 生成（自動 2–3s）→ 課表確認 → 確認儲存後進首頁 → 開始訓練 → 記錄/休息循環 → 完成 → 回首頁。
- **記錄→休息轉場**：按「完成這組」立即切到全螢幕休息倒數（建議滑入/淡入轉場 ~250ms）。倒數每秒遞減，計時環同步收縮（`stroke-dashoffset` 1s linear）。
- **休息結束**：自動切到下一組記錄頁；若是該動作最後一組則進下一動作；若全部完成則進完成頁。
- **編輯彈出層**：點任何動作的「改」chip → 從底部滑入 sheet；步進器即時改值；「完成」寫回資料並關閉；點 sheet 外的遮罩可關閉。
- **編輯會影響訓練**：在課表/開始前改了組數次數休息，開始訓練後記錄頁的目標值即為改後數值。
- **總計時**：訓練開始後持續累計，顯示在記錄/休息頁頂部與完成頁。
- **載入狀態**：AI 生成中頁（進度環 + 逐項勾選）。**錯誤狀態**（生成失敗、斷線）目前未設計，需開發時補上。

## 七、State Management（狀態）

主要狀態變數（以原型邏輯為準）：
- `screen`：當前畫面路由（welcome / body / goal / freq / gen / confirm / home / plan / data / diet / profile / pre / record / rest / finished）。
- `week`：一週課表陣列，每天 `{ day, name, sub, done?, today?, rest?, ex:[{name, w, r, sets, rest}] }`。**這是課表的單一資料來源**，編輯寫回這裡。
- `plan`：開始訓練時從「今天」那天複製出來的工作副本，訓練流程讀它。
- `exIndex` / `setIndex`：目前第幾個動作、第幾組。
- `draftW` / `draftR`：記錄頁當前輸入的重量/次數。
- `log`：每個動作每組實際記錄（`{w, r}`），完成頁的「計畫 vs 實際」由此產生。
- `restRem` / `restTot`：休息剩餘 / 總秒數（計時環比例 = restRem/restTot）。
- `elapsed`：訓練總計時（秒）。
- `editDay` / `editIdx`：編輯彈出層目標（哪一天、哪個動作）；`eS/eR/eRest` 為編輯草稿值。
- `expandedDay`：課表 tab 目前展開的是哪一天。

狀態轉換靠一個每秒的計時器（推進 `elapsed`、休息倒數、AI 生成進度）。

實際產品需補上的資料層：使用者資料、課表持久化、訓練紀錄、體重/飲食紀錄的後端或本地儲存。

## 十、實作建議順序

1. 先搭骨架與導覽（底部分頁 + 路由）。
2. 實作**訓練主場景**（記錄 → 休息計時 → 完成）——這是 App 靈魂，且互動最複雜（計時、背景通知、計畫vs實際）。
